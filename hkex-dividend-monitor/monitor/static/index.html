Ask the assistant

It can manage your watchlist, look up filings already ingested, read a specific filing's full text on demand, and check HKEX directly (up to 3 days) if something's missing entirely. Requires a DeepSeek API key set in Settings. If a reply is wrong, press its 👎 — that saves the full turn (your question, the raw data the tools returned, the reply, and your note) to a log you can download here and feed to an AI coding session to get the cause fixed.

Settings

Saved locally on this computer only (data/settings.json). Keys are only ever sent to the service you configure (DeepSeek / Slack / Discord / Telegram) when needed.

DeepSeek (LLM extraction)

Get a key at platform.deepseek.com → API Keys. The reasoning model is only called when a filing extraction flags itself ambiguous -- routine classify/extract calls always use the faster model above.

Tests the currently saved key -- save settings first if you just changed it.

How do I get a webhook?
Slack: Slack → your workspace → Settings → Apps → search "Incoming Webhooks" → Add to Slack → pick a channel → copy the Webhook URL.
Discord: Server Settings → Integrations → Webhooks → New Webhook → pick a channel → Copy Webhook URL.
Telegram: Message @BotFather → /newbot → copy the token. Then message your new bot once, and visit https://api.telegram.org/bot<TOKEN>/getUpdates in a browser to find your numeric chat ID.
Scheduler
Race mode (arbitrage)

On a target's exact date, the app polls that one ticker directly on HKEX far more often than the normal cycle, and pings you the instant any filing from it appears -- before the AI even reads it.

If race mode can't reach HKEX for a racing ticker, you'll get a "HKEX RACE MODE UNREACHABLE" alert after this many consecutive failures, repeated at most once per cooldown period until it recovers.

Chat assistant

A soft guardrail against unexpectedly high DeepSeek API usage from the Chat tab. Counts messages over a rolling 24 hours.

Bloomberg (blpapi bridge)

Optional. Lets the chat assistant pull live Bloomberg dividend data as a table and adds the same data to dividend alerts. Requires a small bridge script running on the same computer as a logged-in Bloomberg Terminal -- see the README for setup details.
